# TEAM SPIRIT — Universal Text Collector Bot

Автоматический сборщик **всех текстов от 50+ слов** из Telegram-каналов.
Читает публичные каналы **без вступления** (через MTProto API).
Сканирует **ВСЮ историю** от первого до последнего сообщения.

## Возможности

- ✅ Читает публичные каналы **не заходя в них** (Telegram MTProto)
- ✅ Сканирует **все сообщения** от начала до конца (не 200, а все)
- ✅ Сохраняет **ЛЮБОЙ текст от 50 слов** (не только хаудинг)
- ✅ Отслеживает ссылки `t.me/...` в сообщениях — автоматически добавляет в очередь
- ✅ Приватные ссылки (`t.me/+...`) — сохраняет для ручного добавления
- ✅ Скачивает `.txt` файлы и извлекает тексты
- ✅ **Сохраняет прогресс** — если бот перезапустится, продолжит с места остановки
- ✅ Обрабатывает флуд-контроль (FloodWait) — ждёт и продолжает
- ✅ SQLite база + дублирование в `collected_texts.txt`
- ✅ Поиск / статистика / экспорт / управление очередью
- ✅ Работает 24/7 абсолютно бесплатно

## Быстрый старт (5 минут)

### 1. Получить API ключи

- **API_ID и API_HASH**: https://my.telegram.org/apps → Create Application
- **BOT_TOKEN**: @BotFather → `/newbot` → дать имя → получить токен

### 2. Настроить и запустить

```bash
cd /home/radmir/TelegramHaudingCollector

# Установка
pip install -r requirements.txt

# Конфиг
cp .env.example .env
nano .env   # вставить API_ID, API_HASH, BOT_TOKEN

# Запуск (screen — работает после закрытия терминала)
chmod +x start.sh
./start.sh
```

### 3. Пользование

1. Найди бота в Telegram, напиши `/start`
2. Добавь бота в **любой чат** (админом, с правом чтения)
3. Кидай в чат ссылки `t.me/канал` — бот сам добавит их в очередь
4. Бот в фоне сканирует каналы один за другим
5. Все тексты от 50 слов сохраняются

## Как это работает

1. **Бот слушает чаты**, где он состоит
2. Видит ссылку `t.me/канал` → резолвит через MTProto → добавляет в **очередь**
3. **Фоновый процесс** берёт каналы из очереди по одному
4. Для каждого канала: `client.iter_messages(entity, min_id=последний_обработанный, reverse=True)`
5. Каждые 500 сообщений — сохраняет прогресс в БД
6. При флуд-контроле — ждёт указанное время и продолжает
7. При перезапуске — продолжает с последнего сохранённого `msg_id`

**Важно:** бот не заходит в каналы, он их просто читает через API.
Это возможно для **публичных** каналов (у которых есть username).
Приватные (`t.me/+...`) — нужно добавлять бота вручную.

## Бесплатный хостинг 24/7

### Вариант A — Твой ПК (самый простой)

```bash
./start.sh
# Отключиться: Ctrl+A, D
# Вернуться: screen -r hauding
# Остановить: screen -S hauding -X quit
```

### Вариант B — Fly.io

```bash
flyctl auth login
flyctl launch --from Dockerfile
flyctl secrets set API_ID=... API_HASH=... BOT_TOKEN=...
flyctl volumes create collector_data --size 1
flyctl deploy
```

### Вариант C — Render.com

1. Залить на GitHub
2. Render Dashboard → Blueprint → выбрать `render.yaml`
3. Добавить `API_ID`, `API_HASH`, `BOT_TOKEN` в Environment
4. Deploy
